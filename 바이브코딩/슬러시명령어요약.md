# 슬래시명령어요약

* 앤트로픽 공식 문서: https://code.claude.com/docs/ko/slash-commands#bash-명령어-실행
* 제미나이 CLI 문서: https://goddaehee.tistory.com/374

* claude code 명령어
    * /help: 슬래시 명령어 사용법을 안내합니다.
    * /clear: 대화 기록을 삭제합니다.
    * /model: 사용할 AI 모델을 변경합니다. 최신 모델이 나왔으면 한 번씩 확인하시는 것이 좋습니다.
    * /context: 일일 사용량이 정해져있어서 다 쓴 경우에는 일정 시간을 기다려야하는 불편함이 있습니다.
    * /init: CLAUDE.md 파일을 생성, CLAUDE.md 파일을 기준으로 개발하게 됩니다. 그러나 이 파일을 너무 신뢰하지 마십시오.
    * /compact: 대화 내용을 요약합니다. 새로운 대화를 시작할 때 압축한 대화내용을 전달하기 위해서입니다.
    * /rewind: 이전 대화로 돌아갑니다.
    * @ 명령어: 특정 파일이나 폴더를 참고합니다.
    * Alt + M: 수정모드(Yes요청하지 않는 모드), 플랜모드 등을 선택할 수 있습니다.
    * 이미지는 터미널에 드래그앤드롭도 됩니다. Alt + V를 사용하셔도 됩니다.
    * Shift + Enter: 줄바꿈
    * Esc 2번: 입력 취소
    * claude --resume: 이전 대화를 불러옵니다.
    * claude --continue: 이전 대화의 맥락을 이어서 진행합니다.
    * /exit: 종료합니다.
    * /agents: 에이전트 모드를 시작합니다. 역할을 분담해서 개발하게 할 수 있습니다.

# 효율적인 사용 방법

1. 프로젝트 폴더를 만들고 VS Code로 열기
2. 터미널에서 `claude` 실행
3. `/plan`모드로 프로젝트 기획
4. 개발 시작
5. `/init`으로 프로젝트 초기화
6. 필요할 때마다 `/review`로 코드 검토
7. 대화가 길어지면 `/clear` 또는 `/compact` 정리
8. 사용량이 궁금하면 `/context`또는 `/cost`로 확인

# 사용 팁
* 어떤 서비스를 개발할 때에는 미리 이미지, 아이콘, 로고 등을 준비해두면 좋습니다.
* 느낌을 전달하는 것보다는 구체적인 요구사항을 전달하는 것이 좋습니다. 예를 들어서 "apple 스타일에 홈페이지를 개발해줘" 라고 하는 것 보다는 "스타일은 style.json 파일을 참고해줘" 라고 하는 것이 더 정확합니다.
* header, footer, sidebar, 아이콘 등 반복되는 UI 요소는 미리 텍스트로 정리하여 만들어두고 재사용하는 것이 좋습니다. 예를 들어, "아이콘은 icons.md 파일을 참고해줘" 라고 하는 식입니다.
