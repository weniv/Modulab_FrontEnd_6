# 실습하기 전
- 개발하고 싶은 서비스
- 백엔드가 필요한지 체크할 필요가 있습니다.
- 백엔드가 필요하다면 어떤 서비스가 구체적으로 필요한지
- 백엔드 없이 기록만 하는 용도라면 노션이나 구글 시트로도 충분합니다.
- 아래 명령어처럼 '통으로 만드는 명령'은 피하는 것이 좋습니다. 단계별로 여러분이 견고하게 공부를 하게 되실텐데, 그러면 어떤 단계에서 어떤 프롬프트를 작성해야할지 감이 오게 되실겁니다. 그럴 때 앞에서 배운 내용을 토대로 프롬프트를 작성하는 것이 좋습니다.(리라이팅, 템플릿 활용, 템플릿 라이브러리 등)

# 2단계: (저장공간 필요) 솔루션 이용하기(워드프레스, 그누보드, 제로보드 등)

- 솔루션 이기 때문에 커스터마이징의 한계가 있을 수 있습니다. 간단한 서비스라면 이렇게 구현하는 것도 좋습니다.
- 다만 개발자가 되시려고 하시는 분들에게는 권장하지 않습니다.
- 아임웹보다는 커스터마이징이 좀 더 자유롭습니다.
- 결제같은 것은 붙어있지 않습니다.

# 3단계: 백엔드 + 프론트엔드
- 난이도: 100
- 백엔드 언어를 선택해야 합니다.

```
나는 유튜브와 비슷한 동영상 스트리밍 서비스를 간단히 만들려고해. 나는 코딩을 인생에 있어서 한 번도 해본적이 없는 사람이라 사용 방법까지 정리해줘. python은 설치했어. Django라는 백엔드 프레임워크를 사용할거야.

내가 필요한 요구사항은 아래와 같아.

- 유튜브와 비슷한 UI
- 클릭했을 때 영상 재생
- 로그인(로그인한 사용자가 영상을 올릴 수 있는 형태는 아니고, 비공개된 영상을 볼 수 있는 서비스야)
- 나는 admin 페이지에서 영상을 올려, 영상을 올릴 수 있는 사람은 나 혼자야.
- 배포는 render에서 할 거야. 그래서 requirements.txt 파일이 필요하다네! render 서비스에서 배포할 때 처음에 `gunicorn 프로젝트명.wsgi`으로 시작한데! 참고해줘.
```
